① システム全体像

誰が・何を・どこに置いているか — 仕組みのざっくり地図

登場する3つの "場所"

📊 管理SS（RL_CYO 統合管理SS）
全店舗のデータを集約してダッシュボード化する司令塔。
運営者（渉さん）が見るのはここ。GAS スクリプトもここに紐付いている。
🏪 店舗SS（店舗ごと・月ごとに別ファイル）
実際に検品データを記録する個別スプレッドシート。
Caravan：1店舗 = 1ファイル　／　常設：1店舗 × 1ヶ月 = 1ファイル
📷 OCR起動WebApp と 画像フォルダ
オーダーレシートの写真を読み込んで、店舗SSに自動で行データを書き込む仕組み。
画像をアップする読み込みフォルダ と、起動するための Web App URL がある。

データの流れ

📷 オーダーレシートを撮影 → 画像フォルダにアップ
↓
🔍 OCR起動WebApp で OCR を実行（店舗・タブを選択）
↓
📄 店舗SS の検品シートに行データが自動入力
count / name / code / category / color / size / 刺繍 など
↓
👷 作業者が実際の作業（プリント・刺繍など）を行い、判定〇△✕を入力
↓
⏰ 毎日 15:00 / 21:00 にGASが自動集計（または手動で「今すぐ更新」）
↓
📊 管理SS のダッシュボードが更新される
ミス率 / 製作数合計 / 作業者別ミス率 / 送料明細 など
📌 ポイント OCR と ダッシュボードは別系統。OCR は「データの入口」、ダッシュボードは「データの集計と表示」の役割。 OCRが何を読み取ったかではなく、その後の作業品質（プリント・刺繍の出来栄え）に対する判定〇△✕がダッシュボードのミス率になります。

② 新シーズンを始める（Caravan）

例：2026SS / 2026FW など、新しいキャラバンシーズンの立ち上げ手順

🎯 ゴール 管理SS に新しい「【Caravan】2026SS」タブを生成して、店舗データを集計できる状態にする。

手順

1
管理SS のメニューからテンプレ生成
ダッシュボード → テンプレ生成 → Caravan タブ
2
シーズン名を入力
プロンプトに 2026SS のように入力 → 「【Caravan】2026SS」というタブが自動生成される。
3
生成されたタブのレジストリに店舗情報を入力
A〜F列に手動入力：
A列B列C列D列E列F列
店舗名店舗SS ID実施期間開始実施期間終了締め月備考
4/8-14 伊勢丹静岡1_vpZGy9D5g…2026/4/82026/4/144月
4
各店舗SS を用意
シーズン用の検品シート（オーダー入力 + OCRが走る側のSS）を、店舗ごとに用意。
※ 既存テンプレからのコピーで運用。店舗SS の ID をレジストリのB列に貼るのがポイント。
5
ダッシュボードを手動更新（初回確認）
ダッシュボード → 今すぐ更新（特定の期間タブだけ） で 【Caravan】2026SS を選択 → 結果を確認。
6
あとは自動更新トリガーが回り続ける
既存の毎日 15:00 / 21:00 のトリガーが、新しい期間タブも自動で拾います。
※ トリガー未設定なら ダッシュボード → 自動更新トリガー設定
⚠️ 注意 レジストリのB列（店舗SS ID）が間違っていると、その行の集計が エラー 扱いになります。 SS ID は URL の /d/...../edit の .... の部分。コピペミスに注意。
📅

③ 新年度を始める（常設）

例：2027年 など、新しい年の常設プロジェクト立ち上げ手順

🎯 ゴール 常設店舗の 1月〜12月分の店舗SS を全部用意し、管理SS に「【常設】2027」タブを生成して集計できる状態にする。

手順

1
常設店舗SSを年間一括で生成
常設 → 指定年のSSを一括生成
→ 12ヶ月 × 店舗数 の検品SS が一気に作られる。
※ 12月になると、年次トリガーで翌年分が自動生成される設定もあります。
2
管理SS で常設テンプレを生成
ダッシュボード → テンプレ生成 → 常設 タブ
プロンプトに 2027 と入力 → 「【常設】2027」タブが生成される。
3
レジストリに各月のSS IDを貼り付け
A列に店舗名、B〜M列に1月〜12月の店舗SS ID。
A列B列C列...M列N列
店舗名1月 SS ID2月 SS ID...12月 SS ID備考
阪急梅田1-XJC2c…1SGoHD…...1BpQtQ…
4
手動更新で確認
ダッシュボード → 今すぐ更新（特定の期間タブだけ） で 【常設】2027 を選択 → ダッシュボード反映を確認。
5
以降は自動更新でOK
毎日 15:00 / 21:00 の自動更新が、新しい期間タブも対象に。
💡 ヒント 常設は「店舗 × 月」の組み合わせなので、レジストリ入力が一番の作業ボリューム。 SS ID のコピペは 途中で切れない よう注意。
🏪

④ 店舗を追加する（既存シーズン・年度に追加）

シーズン途中で新しい店舗が増えた場合の手順

A. Caravan に店舗追加

1
店舗SSを用意
既存テンプレからコピーして新店舗用のSSを作成
2
レジストリに行追加
該当シーズンタブの A〜F列 最下行に追加
3
手動更新
「今すぐ更新（特定の期間タブだけ）」で確認

B. 常設に店舗追加

1
該当月の店舗SS を用意
月ごとに1ファイル必要
2
レジストリに行追加
A列に店舗名、B〜M列に該当する月のSS ID を貼り付け（途中月から開始可）
3
手動更新
「今すぐ更新（特定の期間タブだけ）」で確認
⚠️ 注意：店舗20件を超える場合 Caravan テンプレートのデフォルト枠は20件。20件を超えると、システムが 自動で行を追加 しますが、 途中で予期せぬ挙動が起きた場合は「テンプレ整合性チェック」で確認してください。

⑤ 日常運用の流れ

毎日 何が動いて、何を確認すれば良いか

登場人物と役割

👷 店舗作業者
OCRで検品シートに入力 → 実作業 → 判定〇△✕を入力（日々）
⏰ 自動トリガー（GAS）
毎日 15:00 と 21:00 に refreshAllDashboards が自動実行
👀 運営者（渉さん）
管理SSのダッシュボードを確認 → 異常があれば店舗SSに飛んで詳細を見る

1日のサイクル（典型例）

🌅 朝〜日中　店舗作業者が検品シートに OCR入力 → 作業 → 判定入力
↓
⏰ 15:00　1回目の自動更新（GASトリガー）
→ 管理SSのダッシュボードが最新状態に
↓
🔍 夕方　運営者がダッシュボードをチェック
↓
⏰ 21:00　2回目の自動更新（夜分の入力分も反映）
↓
📊 翌朝　ダッシュボードを朝イチで確認、必要に応じて手動更新

運営者が確認するポイント

📊
▼ 全体サマリ（常設のみ）
総製作数 / ミス率 / 対象店舗数
📊
▼ 店舗別ミス率
どの店舗のミスが多いか
📊
▼ 作業者別ミス率
作業者ごとのミス傾向
📊
ミス一覧 YYYY
ミスの詳細（商品コード・作業者・日時）一覧
📌 急ぎで反映したい時 自動更新を待たずに、メニュー ダッシュボード → 今すぐ更新（全期間） でいつでも手動更新可能。

⑥ OCR連携の仕組み

オーダーレシートの写真から、検品シートへ自動入力するまでの流れ

💡 Before / After
Before：作業者が検品シートの全列を手入力していた（時間がかかる・タイプミス頻発）
After：レシートを撮ってOCR起動するだけで、商品情報が自動入力（判定だけ人間が入れる）

OCR の6ステップ

1
オーダーレシートを撮影
複数のレシートをまとめて撮るのが効率的（一括処理が基本）
2
読み込みフォルダに画像をアップロード
Google ドライブ内の専用フォルダにアップ。
Caravan：1店舗 = 1フォルダ　 常設：1ヶ月 = 1フォルダ
3
OCR起動 WebApp を開く
管理SS の設定タブにある OCR Web App URL を開く。
ブラウザに UI が表示される。
4
店舗・タブを選択
プルダウンで「どの店舗のどのタブに反映するか」を選択
5
実行 → OCRが画像を解析
画像1枚 = 検品シート1行 として処理
6
検品シートに自動入力される
商品情報・OCR元画像名・更新日時が記録される

⚠️ OCRエラー時 OCRが読み取れなかった項目があると、remark列 に「●●エラー」と自動記載されます。 その場合は手動で訂正 or 再OCRボタン（検品シート内）で再実行できます。
🔗 OCR と ダッシュボードの関係 OCR は 検品シートの入口を効率化するツール。 ダッシュボードのミス率 = OCR の精度ではなく、作業者が入力する判定〇△✕（プリント・刺繍の実作業の品質）に基づきます。 OCR が間違っても、それはダッシュボードのミス率には直接影響しません。

⑦ トラブル時の対応

「あれ、おかしいな」と思った時の診断フロー

症状別 対応マップ

😵 メニューが表示されない
原因：SS のリロードが不足、または onOpen エラー
対応： ① SSタブを 完全に閉じてから開き直す
② それでも出ない場合は GASエディタで 実行ログ を確認
📭 ダッシュボードにデータが入らない
確認手順：
① ダッシュボード → 期間タブ検出テスト
　 → 対象タブが検出されているか
② ダッシュボード → レジストリ&ラベル検出テスト
　 → レジストリ（B列など）が読めているか、ラベルが見つかっているか
③ ダッシュボード → テンプレ整合性チェック
　 → テンプレタブの ▼ ラベルが正しい位置にあるか
🎨 ダッシュボードのレイアウトが崩れる
典型例： ・余り行が残っている　 ・★合計の位置がずれている　 ・列がはみ出している
対応： 手動で ダッシュボード → 今すぐ更新（特定の期間タブだけ） を実行して再描画。
※ 該当タブを一度コピーしてバックアップしてから対処すると安心。
📷 OCR が認識ミスをした
対応：
① 検品シート内の 再OCRボタン から再実行
② 改善しなければ手動で修正（OCRが書いた列を直接編集してOK）
⏱ 自動更新が走っていない
確認：GASエディタの「トリガー」アイコンから登録状況を確認
対応： ダッシュボード → 自動更新トリガー設定 で再登録
⚠️ ストア集計のエラーが出る
典型例： 「店舗SS が見つからない」「アクセス権がない」
対応： ① レジストリの B列（SS ID）にコピペミスがないか
② 該当の店舗SS を直接開いて、アクセスできるか確認
③ SS ID は /d/.../edit の ... 部分のみ

最初に試すべき診断（共通の入口）

1
期間タブ検出テスト
タブが正しく認識されているか
2
レジストリ&ラベル検出
店舗一覧とセクション位置の確認
3
テンプレ整合性チェック
テンプレタブの構造に問題がないか
4
実行ログを見る
GASエディタの「実行数」から
🆘 困った時の最終手段 GASエディタの 変更履歴（左下の時計アイコン）から、過去の動いていた時点のコードに戻すことができます。 また、SSは ファイル → 変更履歴を表示 で過去状態に戻せます。
